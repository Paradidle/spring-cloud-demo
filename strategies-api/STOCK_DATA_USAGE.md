# 股票数据入库与定时任务使用说明

## 功能概述

已实现以下功能：
1. **数据初始化接口** - 一次性拉取所有股票基本信息和当日分时数据并入库
2. **定时任务** - 每5分钟自动拉取分时数据更新到数据库（仅交易时段）
3. **手动触发接口** - 可手动触发分时数据拉取

## API接口

### 1. 初始化数据
**接口**: `POST /api/strategies/init-history`

**功能**: 
- 拉取所有沪深主板股票列表
- 保存股票基本信息到 `stock_basic` 表
- 获取每只股票的当日分时数据
- 保存到 `stock_daily` 表

**返回**: `"历史数据初始化完成"`

**使用场景**: 首次部署或需要全量更新数据时调用

### 2. 手动触发分时数据拉取
**接口**: `POST /api/strategies/fetch-minute-data`

**功能**: 
- 从数据库读取所有股票列表
- 拉取每只股票的5分钟K线数据
- 更新到 `stock_daily` 表的 `minute_data` 字段

**返回**: `"分时数据拉取完成"`

**使用场景**: 测试或需要立即更新数据时调用

## 定时任务配置

### 执行时间
```
cron = "0 */5 9-15 * * MON-FRI"
```
- **频率**: 每5分钟执行一次
- **时段**: 9:00 - 15:00（A股交易时间）
- **日期**: 周一至周五

### 修改定时任务
如需调整执行频率，修改 `StockScheduleServiceImpl.java` 中的 `@Scheduled` 注解：

```java
// 每1分钟执行
@Scheduled(cron = "0 */1 9-15 * * MON-FRI")

// 每10分钟执行
@Scheduled(cron = "0 */10 9-15 * * MON-FRI")

// 每天9:30执行一次
@Scheduled(cron = "0 30 9 * * MON-FRI")
```

## 数据存储说明

### stock_basic 表
存储股票基本信息：
- `stock_code`: 股票代码
- `market`: 市场标识（sh/sz）
- `stock_name`: 股票名称

### stock_daily 表
存储股票日线数据：
- `stock_code`: 股票代码
- `trade_date`: 交易日期
- `open_price`: 开盘价
- `high_price`: 最高价
- `low_price`: 最低价
- `minute_data`: 分时数据JSON数组

**注意**: 
- 同一天同一股票只会有一条记录
- 重复调用会更新现有记录，不会重复插入
- `minute_data` 字段存储完整的5分钟K线JSON数据

## 使用流程

### 首次使用
1. 执行 `sql/stock_tables.sql` 创建数据库表
2. 启动应用
3. 调用 `POST /api/strategies/init-history` 初始化数据
4. 定时任务会自动运行，每5分钟更新分时数据

### 日常使用
- 定时任务自动运行，无需手动干预
- 如需立即更新，调用 `POST /api/strategies/fetch-minute-data`

## 性能优化建议

1. **分批处理**: 如果股票数量很多，可以考虑分批处理
2. **异步处理**: 可以使用 `@Async` 异步执行，避免阻塞主线程
3. **缓存优化**: 可以缓存股票列表，减少数据库查询
4. **错误重试**: 失败的请求可以加入重试队列

## 日志说明

应用会输出以下日志：
- `开始初始化股票数据` - 初始化开始
- `共需要处理XXX只股票` - 股票总数
- `已处理: 基本信息XXX条, 日线数据XXX条` - 处理进度（每50条）
- `数据初始化完成: 基本信息XXX条, 日线数据XXX条` - 完成统计
- `开始定时拉取分时数据` - 定时任务开始
- `分时数据拉取完成: 成功XXX条, 失败XXX条` - 定时任务统计

## 注意事项

1. **数据库配置**: 确保 `application.properties` 中MySQL配置正确
2. **网络连接**: 确保服务器可以访问新浪API
3. **请求频率**: 代码中已添加延迟（100ms/50ms），避免请求过于频繁
4. **交易时间**: 定时任务只在交易日的工作时间执行
5. **事务管理**: 使用 `@Transactional` 保证数据一致性
