# 批量股票箱体分析功能说明

## 功能概述

本功能支持批量分析数据库中所有股票的箱体位置，并将分析结果导出到Excel文件。主要特性包括：

1. **多线程并行处理** - 使用5个线程同时分析多只股票，大幅提升分析速度
2. **断点续传** - 支持中断后从中断处继续分析，避免重复工作
3. **实时进度保存** - 每分析完一只股票就立即保存结果和进度
4. **临时Excel生成** - 每分析50只股票自动生成一次临时Excel文件
5. **详细日志记录** - 实时显示分析进度和百分比

## 使用方法

### 1. 启动批量分析

调用以下API接口启动批量分析任务：

```bash
POST http://localhost:8083/api/strategies/analyze-all-stocks
```

响应示例：
```
批量分析任务已启动，请稍后检查结果
```

### 2. 手动导出Excel（从已有JSON结果）

如果已经有JSON结果文件，可以随时调用此接口生成Excel：

```bash
POST http://localhost:8083/api/strategies/export-excel
```

响应示例：
```
Excel文件已生成: stock_analysis_results/stock_box_analysis_2026-05-12.xlsx, 共 150 个股票
```

**使用场景**：
- 批量分析中断后，想查看已完成的部分结果
- 分析过程中，想随时生成当前进度的Excel
- 分析完成后，重新生成Excel文件

### 2. 查看分析进度

分析过程中会在控制台实时输出进度信息：

```
进度: [50/465] 10.75% - 股票 600000 (浦发银行) 分析完成
进度: [100/465] 21.51% - 股票 600004 (白云机场) 分析完成
...
```

### 3. 查看分析结果文件

所有结果文件都存放在 `stock_analysis_results/` 目录下：

#### 最终Excel文件
- **位置**: `stock_analysis_results/`
- **文件名**: `stock_box_analysis_YYYY-MM-DD.xlsx`
- **内容**: 包含所有已分析股票的完整结果

#### 临时文件（分析过程中生成）
- **临时Excel**: `stock_box_analysis_temp_50_of_465_2026-05-12.xlsx` (每50只股票生成一个)
- **单个股票结果**: `600000.json`, `600004.json` 等（每个股票一个JSON文件）
- **进度文件**: `analysis_progress.txt` (记录已完成的股票列表)

**重要提示**：
- ✅ Excel文件和JSON文件在同一个目录下，方便查找
- ✅ 即使分析中断，也可以调用 `/export-excel` 接口生成部分结果的Excel
- ✅ 每分析完一只股票就立即保存JSON，数据不会丢失

## 断点续传

如果分析过程中断（如程序重启、系统故障等），再次调用API时会自动：

1. 读取 `stock_analysis_results/analysis_progress.txt` 文件
2. 识别已完成的股票
3. 只分析未完成的股票
4. 最终合并所有结果生成完整的Excel

**注意**: 如需重新开始分析，请删除 `stock_analysis_results` 目录。

## Excel文件格式

生成的Excel文件包含以下列（不包含完整分析结果）：

| 列名 | 说明 | 数据类型 |
|------|------|----------|
| 股票代码 | 股票代码，如 600000 | 文本 |
| 股票名称 | 股票名称，如 平安银行 | 文本 |
| 箱顶价格 | AI分析的箱体顶部价格 | 数值 |
| 箱底价格 | AI分析的箱体底部价格 | 数值 |
| 当前价格 | 最新收盘价 | 数值 |
| R值 | 相对位置指标 (0-1之间) | 数值 |
| 位置判断 | 顶部/底部/中部/突破上/突破下 | 文本 |
| 操作建议 | AI给出的一句话操作建议 | 文本 |

**注意**：
- ✅ 所有数值字段都会正确显示，不会出现0的情况
- ✅ 移除了“完整分析结果”列，保持表格简洁
- ✅ 自动调整列宽，方便阅读

## 性能优化

- **线程池大小**: 5个线程（可根据服务器性能调整 `THREAD_POOL_SIZE` 常量）
- **API限流**: DashScope API调用有速率限制，建议使用适中的线程数
- **内存优化**: 单个股票结果立即保存到文件，避免内存占用过大

## 注意事项

1. **API密钥**: 确保在 `application.properties` 中配置了正确的 `spring.ai.dashscope.api-key`
2. **数据完整性**: 分析前确保数据库中已有足够的股票日线数据（至少120天）
3. **磁盘空间**: 分析465只股票约需要100-200MB磁盘空间存储中间文件
4. **分析时间**: 根据API响应速度，分析全部股票可能需要30分钟到2小时

## 故障排查

### 问题1: 分析速度慢
- 检查网络连接
- 降低 `THREAD_POOL_SIZE` 减少并发请求
- 检查DashScope API配额

### 问题2: 内存不足
- 减少 `THREAD_POOL_SIZE`
- 增加JVM堆内存: `-Xmx2g`

### 问题3: 结果文件丢失
- 检查 `stock_analysis_results` 目录是否存在
- 确认有足够的磁盘权限
- 查看日志文件了解具体错误

### 问题4: 断点续传不生效
- 检查 `analysis_progress.txt` 文件是否损坏
- 删除该文件可强制重新分析所有股票

## 常见问题

### Q1: 为什么我没有看到Excel文件？

**A**: Excel文件现在存放在 `stock_analysis_results/` 目录下，与JSON文件在同一位置。如果分析过程中中断，可以调用以下接口手动生成：

```bash
POST http://localhost:8083/api/strategies/export-excel
```

### Q2: 如何查看部分分析结果？

**A**: 有两种方式：
1. 查看临时Excel：`stock_analysis_results/stock_box_analysis_temp_*.xlsx`
2. 手动导出：调用 `/export-excel` 接口生成包含所有已完成股票的Excel

### Q3: 分析中断后如何继续？

**A**: 直接再次调用 `/analyze-all-stocks` 接口，系统会自动跳过已完成的股票，只分析未完成的部分。
