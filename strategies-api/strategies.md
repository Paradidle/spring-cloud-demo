# 箱体识别与价格定位方法（技术分析）

## 1. 目的
给定一段时间的K线数据（至少包含最高价、最低价，或仅有收盘价），自动识别出价格横向震荡形成的箱体区间（即水平支撑位和阻力位），并判断最新价格位于箱体的哪个位置（顶部附近、底部附近、中部，或已突破）。

## 2. 输入数据要求
- **数据格式**：按时间顺序排列的表格，至少包含：
    - 日期（或序号）
    - 最高价（High）
    - 最低价（Low）
    - 收盘价（Close，用于判断当前位置）
- **最小数据量**：至少 20 根K线（推荐 30~60 根）。
- 如果没有最高/最低价，只有收盘价，则改用收盘价识别（精度会下降，文档第5节会区分处理）。

## 3. 箱体识别算法（二选一）

### 3.1 方法一：滚动窗口极值法（最简单）
1. 取最近 `N` 根K线（N默认20，可调整）。
2. 箱顶 = `max(最近N根K线的最高价)`
3. 箱底 = `min(最近N根K线的最低价)`
4. 有效性：`箱顶 > 箱底` 且 `(箱顶 - 箱底) / 箱底 >= 0.02`（振幅至少2%，否则视为无效窄幅盘整）。

### 3.2 方法二：局部极值聚类法（更符合箱体定义，推荐）
**步骤：**
1. **找出所有局部极值点**
    - 局部高点：`High[i] > High[i-1]` 且 `High[i] > High[i+1]`
    - 局部低点：`Low[i] < Low[i-1]` 且 `Low[i] < Low[i+1]`  
      （注意：序列首尾无法判断，跳过）

2. **聚类**
    - 将所有局部高点的价格放入列表 `peaks`，排序。
    - 设定聚类阈值 `threshold = 当前价格区间平均值 * 0.01`（或固定为 0.5% 的价格幅度，例如价格100元时 threshold=0.5）。
    - 遍历排序后的 `peaks`，若相邻两点价格差 ≤ threshold，则归为同一簇；否则新建簇。
    - 对局部低点列表 `troughs` 做相同处理。

3. **选择最大簇**
    - 找出 `peaks` 中点数最多的簇，要求该簇至少包含 **2** 个点。取该簇所有点的算术平均值作为 **箱顶**。
    - 同样处理 `troughs` 得 **箱底**。

4. **有效性检查**
    - 箱顶 > 箱底
    - 箱顶 - 箱底 ≥ 过去20根K线平均真实波幅(ATR) × 0.5（若没法计算ATR，则要求振幅≥2%）。
    - 若无效，则认为当前不存在箱体。
